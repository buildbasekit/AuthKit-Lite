# 🤝 Contributing to Auth Kit (BuildBaseKit)

Thanks for your interest in contributing!

This project is part of **BuildBaseKit** — production-ready backend boilerplates. Contributions are welcome, but should align with the goal of keeping the codebase clean, minimal, and production-focused.

---

## 🚀 Getting Started

1. Fork the repository
2. Clone your fork
3. Create a new branch
4. Run the project locally
5. Make your changes
6. Commit and push
7. Open a Pull Request

---

## 📌 Contribution Guidelines

### ✅ Do

* Keep code **simple and production-ready**
* Follow existing **project structure and naming**
* Write **clear commit messages**
* Add comments where necessary (avoid over-commenting)
* Ensure your code compiles and runs correctly

---

### ❌ Avoid

* Adding unnecessary dependencies
* Over-engineering or complex abstractions
* Breaking existing API contracts
* Committing secrets or sensitive data

---

## 🧪 Testing

Before submitting a PR:

* Test all modified endpoints
* Ensure authentication flow works:

  * Register → Login → Access → Refresh → Logout

---

## 🧱 Project Scope

This project focuses on:

* JWT Authentication
* Role-based Authorization
* Clean Spring Boot architecture

If your contribution is outside this scope, open an issue first to discuss.

---

## 💬 Pull Request Guidelines

* Keep PRs small and focused
* Add a clear description of what you changed and why
* Reference related issues if applicable
